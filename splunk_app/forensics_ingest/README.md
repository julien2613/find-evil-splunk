# forensics_ingest — Technology Add-on (index + sourcetypes)

Versioned TA that defines the `forensics` **index** and the **parsing** of the forensic
sourcetypes. The ingestion itself is done via the official Python SDK
(`../../ingest_to_splunk.py`, `index.attached_socket`).

## Contents
- `default/indexes.conf` — `forensics` index (10-year retention).
- `default/props.conf` — JSON parsing + `_time` extraction:
  - `INDEXED_EXTRACTIONS = json` (fields extracted at index time),
  - `TIMESTAMP_FIELDS = event_time` + `TIME_FORMAT` (the evidence's real time),
  - `MAX_DAYS_AGO = 4000` (the evidence dates from 2018, beyond the default),
  - `KV_MODE = none` + `AUTO_KV_JSON = false` (no double extraction at search time).

## Sourcetype schema

### `forensics:process` (Volatility3 windows.psscan)
| Field | Description |
|---|---|
| `event_time` | timestamp (ISO8601) — source of `_time` |
| `process_id`, `parent_process_id` | PID / parent PID (CIM) |
| `process_name`, `dest` | process name, host (CIM Endpoint.Processes) |
| `create_time`, `exit_time` | creation / exit |
| `threads`, `session_id`, `wow64`, `offset_v` | process metadata |
| `image`, `host_role`, `os` | evidence context |

### `forensics:yara_hit` (YARA-X apt_detection_rules)
| Field | Description |
|---|---|
| `event_time` | acquisition timestamp — source of `_time` |
| `signature`, `dest` | YARA signature name, host (CIM Alerts) |
| `severity` | critical / high / medium / low / informational |
| `mitre` | MITRE ATT&CK technique(s) |
| `description` | rule description |
| `image`, `host_role` | context |

### `forensics:incident` (SOC workflow — `| collect`)
Generated by the scheduled alert (verdict, counts, AI analysis).

## Installation
Copy this folder into `$SPLUNK_HOME/etc/apps/forensics_ingest/` then restart
Splunk (the index-time settings of `props.conf` apply at startup).

> ✅ **CIM-aligned fields**: `forensics:process` follows Endpoint.Processes
> (`process_name`, `process_id`, `parent_process_id`, `dest`); `forensics:yara_hit`
> follows Alerts/IDS (`signature`, `severity`, `dest`). Compatible with Enterprise Security.
