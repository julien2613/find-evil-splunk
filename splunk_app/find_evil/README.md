# Find Evil — Agentic Forensics (Splunk app)

Distributable Splunk app for the *Find Evil* project. Provides the forensic
investigation experience directly inside Splunk Web, with native controls.

## Views

- **AI Investigation** (`ai_investigation`) — dashboard with native Splunk controls
  (analysis-angle selectors + LLM connection, Submit button). Runs the
  **`| ai`** command of the Splunk AI Toolkit on the YARA detections and displays
  the verdict + MITRE kill-chain + recommendations generated by the LLM.
- **Forensic Command Center** (`forensic_command_center`) — Dashboard Studio:
  verdict, critical detections, triage by severity, in-memory LOLBins,
  timeline, MITRE table.

## Prerequisites

- `forensics` index populated (see `../../ingest_to_splunk.py`).
- For *AI Investigation*: Splunk AI Toolkit configured with an LLM connection
  named `claude` (see `../../AI_TOOLKIT_SETUP.md`).

## Installation

Copy this folder into `$SPLUNK_HOME/etc/apps/find_evil/` then restart Splunk
(or reload). Access: `http://localhost:8000/en-US/app/find_evil/ai_investigation`.

## Packaging

```bash
cd splunk_app && tar czf find_evil.spl find_evil/   # installable Splunkbase package
```

## Automated SOC workflow

`savedsearches.conf` defines the **Find Evil - Auto Triage Workflow** alert (scheduled
every 30 min): detects the critical YARA detections → AI triage via `| ai` →
writes an incident (`sourcetype=forensics:incident`) shown in the **SOC Incidents** view.

⚠️ The saved search must **belong to a user who has `apply_ai_commander_command`**
(`mltk_admin` role) for `| ai` to run in the scheduled context:
```bash
curl -sk -u <user:pass> -X POST \
  "https://localhost:8089/servicesNS/nobody/find_evil/saved/searches/Find%20Evil%20-%20Auto%20Triage%20Workflow/acl" \
  -d owner=<user> -d sharing=app
```
